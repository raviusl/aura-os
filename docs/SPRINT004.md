# Sprint 004

## Goal

Design the **complete User Journey** for RIVA OS across ten roles.

## Objectives

1. Define journeys for Owner, Admin, Event Planner, Coordinator, Finance, Sales, Designer, Vendor, Client, and Guest (View Only).
2. For each role, document responsibilities, dashboard, modules, daily workflow, permissions, common actions, notifications, and AI assistance.
3. Provide detailed user flow diagrams in Mermaid.
4. Align journeys with [DATA_MODEL.md](./DATA_MODEL.md) and [ROADMAP.md](./ROADMAP.md).

## Deliverables

| File | Description |
| --- | --- |
| `docs/USER_JOURNEY.md` | Full user journey specification |
| `docs/SPRINT004.md` | This sprint summary |

## Explicit non-goals

- Do **not** generate application code
- Do **not** modify the database or migrations
- Do **not** modify UI

## Exit criteria

- All ten roles documented with the eight required dimensions per role
- Cross-role workflows illustrated in Mermaid
- Module access matrix and notification framework defined
- Documentation linked from `docs/README.md`

## Context

- **Sprint 003** defined the core data model (Workspace, Event, Files, Viewer roles).
- **Sprint 004** translates that model into role-specific experiences before Phase 1 feature depth is implemented.
