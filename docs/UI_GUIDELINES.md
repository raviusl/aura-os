# UI Guidelines

Design principles for RIVA OS. Product UI should feel like a premium operating system for planners — calm, fast, and intentional.

## Principles

### Luxury

The product should feel high-end without ornament. Prefer refined materials, restrained color, and polished detail over decorative effects. Every screen should feel considered.

### Minimal

Reduce chrome and cognitive load. One primary action per view when possible. Prefer whitespace and hierarchy over borders, cards, and dense panels. If a visual element does not aid understanding or interaction, remove it.

### Apple-like

Clarity, consistency, and craft. Interactions should feel obvious. Typography and spacing do the heavy lifting. Avoid noisy gradients, glow stacks, and novelty UI patterns.

### Fast

Perceived and real performance matter. Keep interactions snappy, loading states light, and navigation immediate. Prefer progressive disclosure over heavyweight first paints.

### Mobile-first

Design for small screens first, then expand. Touch targets, readable type, and single-column flows come before dense desktop layouts. Desktop should feel spacious — not like a stretched phone UI with extra clutter.

### Soft animation

Motion should be subtle and purposeful: gentle fades, short transitions, soft layout shifts. Use animation to reinforce hierarchy and presence, not to decorate. Avoid bounce-heavy or attention-grabbing effects.

### Large spacing

Generous padding and vertical rhythm. Let content breathe. Prefer larger section gaps and clear grouping over tightly packed widgets. Density is earned, not default.

### Premium typography

Use expressive, purposeful type. Strong hierarchy between display, title, body, and meta. Avoid generic system defaults as the brand voice. Keep line length comfortable and labels concise.

## Practical defaults

- Prefer composition over dashboard clutter on primary screens
- Prefer large spacing and clear type hierarchy over cards and borders
- Prefer soft motion (2–3 intentional moments) over constant micro-animation
- Prefer mobile-readable layouts before multi-column density
- Prefer quiet luxury over trend-driven purple/glow aesthetics
